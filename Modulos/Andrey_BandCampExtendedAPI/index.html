BandCamp API Extendido: busqueda 'Tags'

BandCamp ofrece un API para desarrolladores bastante completo, pero actualmente esta deshabilitado y no posee un módulo para búsqueda por tags. Por esta razon es que he desarrollado una servicio web que se alimenta del HTML de BandCamp y de esta forma se facilita el proceso de desarrollo.

http://bandcamp.com/developer

Ejemplo BandCamp API:
http://api.bandcamp.com/api/band/3/info?key=vatnajokull&band_id=203035041&debug

La utilización de este servicio es libre para los estudiantes del curso "Lenguajes de Programación" del ITCR, de lo contrario por favor ponerse en contacto.

¿Qué es un JSON?

La mayoría de los API's devuelven repuestas en formato XML o JSON. El API de BandCamp devuelve respuestas en formato JSON y por el momento este servicio solo soporta respuestas en formato JSON también.

{ "band_id": 203035041, "track_id": 2323108455 }

Para más información de este formato, consultar la página de Wikipedia JSON o la fuente oficial del formato JSON.org.

¿Como usarlo?

Siguiendo la misma estructura del BandCamp API, la consulta básica:

http://api.andreybolanos.com/bandcamp/?m=tag&f=search&name={TAG}

Debe remplazarse {TAG} por el tag buscado (países, generos, artistas, etc.)

Parametros

Configuraciones del API

Cada parametro se agrega como &{PARAMETRO}={VALOR}
Parametro Descripción Valores Req
m Nombre del modulo
** en desarrollo		 m = tag SI
f Nombre de la funcion
** en desarrollo		 f = search SI

Configuraciones del modulo TAG :: funcion SEARCH

http://api.andreybolanos.com/bandcamp/?m=tag&f=search[parametros..]

Los siguientes parametros afectan el rendimiento de la consulta!

Cada parametro se agrega como &{PARAMETRO}={VALOR}
Parametro Descripción Valores Req
name Nombre del tag string SI
incAlbumData Incluye los datos del album en
results > data > album_data
Default: 0 (false)		 0 (false) | 1 (true) Opcional
incBandData Incluye los datos de la banda en
results > data > band_data
Default: 0 (false)		 0 (false) | 1 (true) Opcional
incTrackData Incluye los datos de la cancion en
results > data > track_data
Default: 0 (false)		 0 (false) | 1 (true) Opcional
limit Limita la cantidad de resultados
Default: 10		 int Opcional
